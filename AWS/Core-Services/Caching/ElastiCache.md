## ✅ Amazon ElastiCache (Exam Essentials)

- **Type**: Managed in-memory caching service.
- **Supports**: Redis and Memcached engines.
- **Scope**: Regional, deployed in Multi-AZ for HA (High Availability).
- **NOT serverless**: You must provision and manage nodes (similar to RDS).
- **Data model**: Key-Value pairs (fast retrieval from memory).
- **Use case**: Improve application performance by reducing database load.

## ✅ One-liner for exam:
- **ElastiCache** = Managed in-memory cache (Redis/Memcached), NOT serverless, stores key-value pairs for ultra-fast access.
