A common migration pattern: **map on-premises technology to equivalent AWS managed services** for easier transition.

***

### ✅ **Configuration Management**

*   **On-premises:** Chef, Puppet
*   **AWS equivalent:** **AWS OpsWorks** (supports Chef and Puppet)

***

### ✅ **File Storage**

*   **On-premises:** Windows File Server
*   **AWS equivalent:** **Amazon FSx for Windows File Server**
*   For Linux-based file systems: **Amazon FSx for Lustre**

***

### ✅ **Message Broker**

*   **On-premises:** ActiveMQ, RabbitMQ, MQTT, AMQP
*   **AWS equivalent:** **Amazon MQ** (supports ActiveMQ and RabbitMQ)

***

### ✅ **Relational Databases**

*   **On-premises:** Oracle, MySQL, PostgreSQL, SQL Server
*   **AWS equivalent:** **Amazon RDS** (supports multiple engines) or **Amazon Aurora** (MySQL/PostgreSQL-compatible)

***

### ✅ **Data Warehouse**

*   **On-premises:** Teradata, Oracle Exadata
*   **AWS equivalent:** **Amazon Redshift**

***

### ✅ **Big Data Processing**

*   **On-premises:** Hadoop, Spark clusters
*   **AWS equivalent:** **Amazon EMR**

***

### ✅ **Identity Management**

*   **On-premises:** Active Directory
*   **AWS equivalent:** **AWS Directory Service (AD)** or **AWS Single Sign-On**

***

### ✅ **Container Orchestration**

*   **On-premises:** Kubernetes
*   **AWS equivalent:** **Amazon EKS** (managed Kubernetes) or **Amazon ECS**
