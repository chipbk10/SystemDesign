**7 migration strategies** ("7 Rs")

*   **Retire**: Decommission or archive applications that no longer provide business value and shut them down. [\[docs.aws.amazon.com\]](https://docs.aws.amazon.com/prescriptive-guidance/latest/large-migration-guide/migration-strategies.html)
*   **Retain**: Keep applications running on-premises (or wherever they are) if you’re not ready to migrate them yet. [\[docs.aws.amazon.com\]](https://docs.aws.amazon.com/prescriptive-guidance/latest/large-migration-guide/migration-strategies.html)
*   **Rehost**: Lift and shift your application to AWS with minimal or no modifications. [\[docs.aws.amazon.com\]](https://docs.aws.amazon.com/prescriptive-guidance/latest/large-migration-guide/migration-strategies.html)
*   **Relocate**: Move servers or applications to AWS at scale with minimal architecture changes. [\[docs.aws.amazon.com\]](https://docs.aws.amazon.com/prescriptive-guidance/latest/large-migration-guide/migration-strategies.html)
*   **Repurchase**: Replace your existing application with a SaaS or cloud-based solution. [\[docs.aws.amazon.com\]](https://docs.aws.amazon.com/prescriptive-guidance/latest/large-migration-guide/migration-strategies.html)
*   **Replatform**: Shift to the cloud while making optimizations like switching to managed databases or newer OS versions. [\[docs.aws.amazon.com\]](https://docs.aws.amazon.com/prescriptive-guidance/latest/large-migration-guide/migration-strategies.html)
*   **Refactor (Re‑architect)**: Rebuild or modernize the application using cloud-native architectures for improved scalability and agility. [\[docs.aws.amazon.com\]](https://docs.aws.amazon.com/prescriptive-guidance/latest/large-migration-guide/migration-strategies.html)
